# jpstories Subagent Translation Sheet Worker

Your task is to fill the empty produce blocks in assigned jpstories plain-text translation sheet files.

Each input file is generated by `go run ./cmd/jpstories export-agent-work -story <story>` from a canonical JSON work item. The sheet contains metadata, sentence IDs, a pre-filled source-context block (read-only), and empty produce blocks. The `levels:` metadata line lists exactly which blocks you must fill. Fill those empty blocks, then write the completed sheet to the assigned output directory, normally `stories/<story>/agent-done/`, using the same filename.

Work only from the assigned input sheet files and this skill. Write only the completed output sheet files and any validation-driven fixes to those outputs. Do not write JSON; the coordinator imports sheets back into JSON with `import-agent-work`.

## Assignment Inputs

Your prompt should provide:

- one or more input sheet file paths, normally from `stories/<story>/agent/`
- an output directory, normally `stories/<story>/agent-done/`
- optionally, a note about which story or chunk is being assigned

Read only the assigned input files. Write completed sheet files to the output directory with the same filenames.

## Sheet Format

The **input** file (`agent/*.txt`) contains the source context and empty produce blocks. The pre-filled block is the read-only source — do not include it in your output. For an English-source story the source block is `english:`; for a Japanese-source story the source block is `native:`.

English-source example (produce `native`, `n3`, `n3_abridged`):
```text
## p-001 / s-001
english:
<<<JPSTORIES
The English sentence appears here.
JPSTORIES>>>
native:
<<<JPSTORIES

JPSTORIES>>>
n3:
<<<JPSTORIES

JPSTORIES>>>
```

Japanese-source example (produce `english`, `n3`, `n3_abridged`):
```text
## p-001 / s-001
native:
<<<JPSTORIES
駅で小さな鐘が鳴った。
JPSTORIES>>>
english:
<<<JPSTORIES

JPSTORIES>>>
n3:
<<<JPSTORIES

JPSTORIES>>>
```

Your **output** file (`agent-done/*.txt`) uses a translation-only format. Write it like this:

```text
# jpstories translation output v1
story_id: <story_id>
chunk_id: <chunk_id>
levels: <comma-separated levels from the source sheet>

## s-001
<level>:
<<<JPSTORIES
<translation>
JPSTORIES>>>
...

## s-002
...
```

Key differences from the input format:
- Header line: `# jpstories translation output v1`
- Metadata: only `story_id`, `chunk_id`, `levels` (no `story_title`, `source_file`, `source_language`, etc.)
- Sentence headers: `## s-XXX` only (no paragraph ID)
- **Do not include the source-context block** (the pre-filled block) in your output
- Only the produce blocks listed in `levels:` go in the output

## Hard Rules

- Write only the output format described above to `agent-done/`.
- Fill every requested produce block (one per sentence ID per level listed in `levels:`).
- Do not re-output the pre-filled source-context block.
- Do not add, remove, rename, reorder, or skip sentences from the source.
- Do not add notes, commentary, Markdown fences, extra metadata, or extra fields.
- Save plain text only, using the output format shown above.
- If your stream is interrupted, content-filtered, or you cannot finish every assigned block, report the file as incomplete instead of success.

## Produce Block Guidance

Use each translation block label exactly as it appears in `levels:`.

### native

Write natural Japanese. Preserve meaning, tone, character voice, and narrative flow. Do not simplify just because the project is for learners. Avoid translationese when a natural phrasing is available.

### n3

Write learner-friendly Japanese around JLPT N3 difficulty. Prefer common vocabulary and direct sentence structure. Avoid dense literary phrasing, uncommon compounds, and highly idiomatic expressions. Preserve important meaning even when simplifying grammar. Keep the result natural, not a word-by-word gloss.

### n3_abridged

Write a shorter JLPT N3-level version. Preserve essential events, relationships, and emotional meaning. Compress details when useful, but do not change the plot. Use Japanese suitable for beginner-intermediate learners. Split longer sentences with multiple clauses into shorter sentences when that improves readability.

### english

When the source is Japanese (`native:` block), produce a **literal English translation**: follow the Japanese sentence structure and word choices closely rather than polishing into natural English prose. Translate individual sentences with no cross-sentence context. Do not add interpretation beyond what the source explicitly states. This literal style is intentional for language-learning purposes.

## Workflow

For each assigned file:

1. Read the input sheet.
2. Before editing, make a short private checklist of the expected sentence headers and requested block labels in file order.
3. Fill only empty translation blocks.
4. Write the completed sheet to the output directory with the same filename.
5. Re-read the completed sheet and compare it against the checklist: every expected sentence header is present in the same order, every expected translation block label is present once, and no new label was added.
6. Check that every requested translation block is non-empty after trimming whitespace.
7. Check that all metadata, English blocks, field labels, and fences are still present and unchanged.
8. Report the exact output filenames so the coordinator can gate only those files.

Do not report success until every assigned output file passes your own checklist. If one file fails, repair that file first and report the unresolved issue only if you cannot make it pass.

If you are retrying a failed file, work only on that one completed sheet and compare it against the matching source sheet. Do not rewrite neighboring files. If the prior output is partial or badly malformed, say so plainly so the coordinator can quarantine it and restart from the original source sheet.

## Self-Check Checklist

Use this checklist before your final report:

- Same filename in `agent-done/` as the input filename in `agent/`.
- Output header is `# jpstories translation output v1`.
- Metadata lines present: `story_id`, `chunk_id`, `levels` (matching the source sheet values).
- Every sentence ID from the source sheet appears exactly once in the output, in the same order.
- Each requested label from `levels` appears exactly once per sentence.
- The pre-filled source-context block is NOT included in the output.
- No unsupported labels or notes were added.
- Every requested produce block is non-empty.
- Every block has both `<<<JPSTORIES` and `JPSTORIES>>>` fences.

The coordinator will gate your assigned files before marking the batch complete:

```powershell
go run ./cmd/jpstories validate-agent-work -story <story> <assigned-sheet.txt>
```

If the coordinator reports a validation error, repair only the failed completed sheet while preserving the same format. After all worker batches pass this gate, the coordinator imports sheets back into JSON and validates again before merging.

When a file contains multiple requested levels, translate all assigned levels in one pass over the source context. Do not copy a translation from one level into another without adapting it to that level's rules.

## Final Report

Briefly report completed files, skipped files, and unresolved issues. Do not paste full sheet contents into chat.
If any assigned file is incomplete because of an interruption, content-filter stop, or uncertainty about the sheet structure, list it under unresolved issues rather than completed files.
